# Projects Repository
This repository contains a collection of security, automation, and engineering projects I have built outside of CTF walkthroughs and vulnerability research. These projects represent practical work in identity security, automation, security awareness, detection engineering, and enterprise-scale problem solving.

The goal of this repository is to provide clear, technical demonstrations of my capabilities as a security engineer, IAM practitioner, and program builder.

---

## Repository Structure
Each project is contained in its own directory with a dedicated README describing:
- The problem it solves  
- The technical approach  
- Tools, languages, and platforms used  
- Architecture or workflow details  
- Key outcomes or measurable impact  

This repository excludes:
- CTF walkthroughs  
- Vulnerability discoveries  
- Any content containing sensitive or proprietary information  

Those items are kept separate to maintain ethical and professional boundaries.

---

## Project Categories

### Identity & Access Management (IAM)
Projects focused on identity lifecycle automation, access governance, and enterprise identity engineering. Examples include:
- Automated AD permissions auditing  
- Password blocklist enforcement  
- Identity lifecycle workflows integrating HRIS and directory services  
- Okta or Azure AD automation scripts  

### Security Automation & Engineering
Tools and scripts designed to reduce manual workload, improve detection, or enforce security controls. Examples include:
- Automated password auditing pipelines  
- Log parsing and enrichment tools  
- API-driven security workflows  
- PowerShell and Python utilities  

### Security Awareness & Behavioral Security
Projects related to user-focused security, training design, and behavioral analysis. Examples include:
- Security Awareness Training (SAT) program documentation  
- Phishing simulation workflows  
- UX-informed interview and feedback processes  
- Reporting and KPI analysis frameworks  

### Detection & Response
Projects that support detection engineering or incident response. Examples include:
- Indicators-of-compromise (IOC) enrichment scripts  
- Log analysis tooling  
- Detection logic prototypes  
- Response workflow automation
