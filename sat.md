# Security Awareness Training (SAT) Program  
Enterprise-Scale Behavioral Security Program | Phishing Simulation | User Risk Reduction

## Overview
This repository documents the Security Awareness Training (SAT) Program I designed and operated at Raymour & Flanigan. The program served more than 5,000 associates and combined structured training, simulated phishing, user behavior analysis, and ethical review processes to build a measurable, sustainable security culture.

The program was launched when I stepped into the Security Awareness Trainer role, and I was responsible for designing the monthly training cycle, developing phishing simulations, analyzing KPIs, and maturing the program over time.

---

## Program Objectives
- Strengthen user resilience against social engineering attacks  
- Deliver regular, digestible micro-trainings aligned with real-world threats  
- Measure and improve phishing susceptibility  
- Build a positive, ethical, user-centric security culture  
- Provide leadership with clear, actionable KPIs  
- Continuously evolve difficulty as user performance improves  

---

## Monthly Training Cycle

### 1. Training Assignment
At the start of each month, new training modules were assigned based on:
- Current threats in the retail sector  
- Internal incidents or trends  
- Results from prior phishing campaigns  
- Optional “free-play” modules based on user interest  

Training content included videos, interactive modules, and scenario-based lessons delivered through the SAT platform.

### 2. Simulated Phishing Campaign
Simulated phishing campaigns were delivered mid- to late-month and followed a structured process:
- Multi-round internal review  
- Ethical constraints (no sensitive or emotionally manipulative themes)  
- Realistic templates aligned with current threat actor behavior  
- Campaign duration based on baseline testing  
- Automatic assignment of corrective training for users who interacted with phishing emails  

### 3. User Feedback Interviews
Using principles from Human-Computer Interaction (HCI) and UX research, I developed a structured interview process:
- Randomly selected users who failed the phish  
- Voluntary participation  
- Interviews focused on:  
  - Why the user fell for the phish  
  - Whether corrective training was helpful  
  - What cues they missed  
  - Suggestions for improving training  
- This process was also used for post-incident investigations after real social engineering events  

### 4. Reporting
At the end of each cycle, I produced a monthly report for leadership that included:
- Participation rates  
- Phish fail/pass rates  
- User vulnerability metrics  
- Trend analysis  
- Recommendations for the next cycle  

This report was delivered prior to the broader monthly security report.

## Program Outcomes

### Initial Challenges
- Early buy-in required collaboration across multiple departments  
- Initial KPIs were below target  
- Needed to demonstrate program value to leadership and associates  

### Improvement and Maturity
Within three months:
- KPIs met or exceeded goals  
- User vulnerability decreased  
- Participation increased  
- Training engagement improved  

### High Performance
Several simulated phishing campaigns resulted in:
- Top-percentile performance compared to the retail industry  
- Significant reduction in high-risk user behavior  
- Increased engagement with optional training modules  

---

## Ethical and User-Centric Design
The program followed strict ethical guidelines:
- No use of sensitive or emotionally charged themes  
- Respectful, non-punitive corrective training  
- Transparent communication with leadership  
- Interviews designed to understand behavior, not assign blame  

This approach increased trust and long-term participation.
